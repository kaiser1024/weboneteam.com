Chromebook to Linux

The precise problem definition: can you modify a Chromebook to behave like a standard UEFI-based x86 PC in order to enable Linux as the OP?

Sidenotes:

  • Does enabling stardard UEFI-based x86 behavior directly and simultaneously enable both Linux and Windows OS installation?
  • Swapping Chromebook firmware (coreboot + Depthcharge) to UEFI is necessary but not sufficient to enable Linux: OS installation relies on understanding EFI executables + GTP as well as allowing the booting of EFI bootloaders

Typical PC setups:

1. EC → UEFI → bootloader → Windows/Linux (dominant PCs)

2. EC → BIOS → bootloader → Linux/Windows (older PCs)

3. EC → Apple firmware → boot → macOS/Linux (Macs)

4. EC → coreboot → Depthcharge → ChromeOS/Linux (Chromebooks)

5. EC → coreboot → UEFI → bootloader → Linux (modified Chromebooks)

Background

Challenges in replacing ChromeOS with Linux on Chromebook

  • Chromebox has no UEFI (approximate equivalent of UEFI on stock Chrome: coreboot + Depthcarge)
  • stock Chromebox has no legacy BIOS as per factory configuration

Key questions

Can you replace Chromebook firmware with UEFI?

Answer: No, Chromebook as a piece of hardware only works with Chromebook firmware which is coreboot. UEFI cannot replace coreboot.

Does coreboot allow booting Linux as the Operating System (OS)?

Answer: No, coreboot cannot load Linux directly.

Can coreboot be used as a starting point to load Linux?

Answer: Yes, it is possible to install an additional layer on top of coreboot which is EDK II (UEFI) to be loaded as coreboot payload.

What are the conditions to load Linux (or any Operating System)?

Answer: hardware → EC powers system → CPU starts execution (reset vector) → low-level initialization runs → RAM becomes usable → devices become accessible → boot environment is established → OS loader is invoked → OS loads
